1-1、basebutton
getImgUrl中${name}.svg加入svg在button元件中會變破圖，但在匯入到其他元件裡是正常的；若去掉svg，則變成button元件裡正常，匯入的其他元件裡是破圖
=> 發現是basebutton.stories.ts的args icon設定成figma.svg，但在basebutton.vue抓取路徑已經有寫了svg，所以系統抓的是figma.svg.svg，導致抓不到圖，改成figma就好了


1-2、lifecard照片匯入出現破圖
const getImgUrl = (name: string) => {
    // 原先寫法: return new URL(`../assets/pic/life/${name}.jpeg`, import.meta.url).href;
    // 但系統抓取不到顯示404 undfineds => Vite 有時候會因為太過「精簡」，導致在靜態分析時沒辦法確定基礎路徑起點在哪裡，特別是在 Storybook 這種多層環境下，它解析出來的路徑就會變成undefined`。
    // 明確地把路徑字串先賦值給變數，提高編譯器成功解析「相對路徑」的機率
    const path = `../assets/pic/life/${name}.jpeg`;
    return new URL(path, import.meta.url).href;
};

1-5、projectcard也找不到圖
使用v-if="item"但顯示找不到item
=> 在Project.stories.ts裡面加入
import { Project } from '@/data/projectcard';
然後在args加入item: Project[0], 就顯示了

? 為什麼在ProjectCard.stories.ts引入{ProjectItem}跟補上item: Project[0]就顯示了?原理是什麼?
但lifecard.stories.ts並沒有寫入東西也能顯示
=> 兩者條件引用的不同，
Project 用props引入data，且用v-if條件渲染，在沒有args傳入item，那item就是undefined，加入item:project[0]，就會有具體的資料
LifeCard是直接引入data，沒有透過defined props的形式匯入，所以不管有沒有傳資料進來，都會自己去抓取資料
